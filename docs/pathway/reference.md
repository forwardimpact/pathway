## Co-located File Structure

All entity files (behaviours, disciplines, tracks) contain both human and agent
definitions in the same file. Skills are embedded within capability files.

### Capability File (with embedded skills)

Skills are defined within capability files, grouped with related checklists and
responsibilities:

```yaml
# capabilities/scale.yaml
name: Scale
emojiIcon: "ğŸ”§"
displayOrder: 2
description: Architecture, quality, and technical sustainability...
transitionChecklists:
  plan_to_code:
    working:
      - Technical approach is documented
      - Dependencies are identified
professionalResponsibilities:
  awareness: Follow established patterns with guidance...
  working: Design components and services independently...
managementResponsibilities:
  awareness: Track technical quality metrics...
skills:
  - id: code_quality
    name: Code Quality & Review
    isHumanOnly: false # Optional, defaults to false
    human:
      description: Writing and reviewing clean, maintainable code...
      levelDescriptions:
        awareness: You follow team coding conventions...
        foundational: You write readable, well-structured code...
        working: You produce consistently high-quality code...
        practitioner: You establish and enforce quality standards...
        expert: You shape coding standards across the business unit...
    agent:
      name: code-quality-review
      description: |
        Guide for reviewing code quality. Use when reviewing PRs,
        conducting code reviews, or assessing code health.
      body: |
        # Code Quality & Review
        ## When to use this skill
        ...
  - id: architecture_design
    name: Architecture & Design
    human:
      description: Designing software systems...
      levelDescriptions:
        awareness: You understand basic patterns...
        expert: You shape architectural standards...
    agent:
      name: architecture-design
      description: Guide for system design...
      body: |
        # Architecture & Design
        ...
```

### Discipline File

```yaml
# disciplines/software_engineering.yaml
specialization: Software Engineering
roleTitle: Software Engineer
description: Builds and maintains software systems...
coreSkills:
  - architecture_design
  - code_quality
supportingSkills:
  - devops
  - cloud_platforms
broadSkills:
  - stakeholder_management
behaviourModifiers:
  systems_thinking: 1
human:
  professionalRoleSummary: We are seeking a skilled {roleTitle}...
  managerRoleSummary: We are seeking an experienced {specialization} leader...
agent:
  coreInstructions: |
    You are a {roleTitle} agent...
  constraints:
    - Do not skip tests
```

### Behaviour File

```yaml
# behaviours/systems_thinking.yaml
name: Think in Systems
human:
  description: The ability to see beyond individual components...
  maturityDescriptions:
    emerging: Recognizes that systems have interconnected parts...
    exemplifying: Defines organizational systems architecture principles...
agent:
  title: Consider the whole system
  principles: |
    - Identify upstream and downstream impacts of every change
    - Consider how changes affect non-functional requirements
  workflow: |
    For every change:
    1. Identify upstream and downstream impacts
    2. Consider non-functional requirements
    3. Document assumptions and trade-offs
```

### Track File

```yaml
# tracks/platform.yaml
name: Platform
description: Internal tooling and infrastructure focus...
roleContext: |
  You operate in a platform context, building tools and
  infrastructure that other engineers depend on.
professional: true
management: false
skillModifiers:
  scale: 1
  documentation: 1
agent:
  coreInstructions: |
    You are a Platform {roleTitle} agent. Your primary focus is 
    building self-service capabilities that enable other engineers...
  constraints:
    - Maintain backward compatibility
```

## CLI Commands

| Command                             | Description                              |
| ----------------------------------- | ---------------------------------------- |
| `npx pathway discipline`            | Browse disciplines                       |
| `npx pathway grade`                 | Browse grades                            |
| `npx pathway track`                 | Browse tracks                            |
| `npx pathway behaviour`             | Browse behaviours                        |
| `npx pathway skill`                 | Browse skills (summary, list, or detail) |
| `npx pathway driver`                | Browse drivers                           |
| `npx pathway stage`                 | Browse lifecycle stages                  |
| `npx pathway tool`                  | Browse recommended tools                 |
| `npx pathway job <d> <t> <g>`       | Derive job definition                    |
| `npx pathway interview <d> <t> <g>` | Generate interview questions             |
| `npx pathway progress <d> <t> <g>`  | Analyze career progression               |
| `npx pathway questions`             | Browse interview questions               |
| `npx pathway agent <d> <t>`         | Generate agent profiles                  |
| `npx pathway --validate`            | Validate all data files                  |
| `npx pathway --generate-index`      | Generate browser index files             |

### Output Modes

| Mode    | Flag      | Description                         |
| ------- | --------- | ----------------------------------- |
| Summary | (default) | Concise overview with statistics    |
| List    | `--list`  | IDs only, one per line (for piping) |
| Detail  | `<id>`    | Full entity information             |
| JSON    | `--json`  | Machine-readable output             |

## Job Description Template

Job descriptions are generated from `Discipline Ã— Track Ã— Grade`:

| Section          | Source                                                |
| ---------------- | ----------------------------------------------------- |
| Title            | `{Discipline.roleTitle} {Grade.title} - {Track.name}` |
| Role Summary     | Discipline + Track + Grade expectations               |
| Responsibilities | Categories at max skill level                         |
| Behaviours       | Behaviour profile at derived maturity                 |
| Skills by Level  | Skill matrix grouped by level                         |
| Qualifications   | Grade qualification summary                           |

## Agent Profile Template

| ------------------- | ------------------------------------------------- |

| Core Identity       | Track or discipline coreInstructions + top skills |
| Operational Context | Track roleContext                                 |
| Working Style       | Top behaviours with workflow patterns             |
